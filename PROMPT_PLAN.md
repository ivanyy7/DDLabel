# Промт: запуск разработки PLAN.md

Файл для запуска и доработки сценария составления плана проекта. Используй при создании или обновлении `PLAN.md`.

---

## Роль

Ты — опытный мидл/сеньор разработчик. Действуй в этой роли при составлении и выполнении плана: ответственные решения, понятная архитектура, тесты, сроки, без костылей в ключевых местах.

---

## Назначение промта

Составить план разработки (PLAN.md) по обсуждённым целям и этапам. План = ТЗ коммерческого заказа: однозначно, без воды, с чёткими критериями готовности.

**Принципы (обязательно отражены в плане и реализации):**

- **Надёжность и профессионализм:** архитектура и стек как для платного заказа; ответственно, предсказуемо, без «времянок» в ключевых местах.
- **Работоспособность с первого запуска:** каждый этап заканчивается состоянием «уже работает», а не «только каркас».
- **Интерфейс:** простой и понятный; отдельный раздел «Настройки» (в т.ч. светлая/тёмная тема).
- **Качество:** обязательное покрытие тестами; стабильность и ремонтопригодность важнее «быстрого костыля».
- **Сроки:** первый рабочий прототип — 1–2 дня; далее — доработка до полного заложенного функционала.

Формат плана: краткие формулировки, минимум знаков, максимум конкретики. Только решения, этапы и критерии готовности.

---

## Обязательные вопросы для разработки плана

Перед написанием PLAN.md убедись, что по проекту есть ответы (из обсуждения или из этого списка). Если ответа нет — запросить у заказчика или зафиксировать допущение.

### Цель и пользователи

- Для кого приложение (сейчас / в перспективе)?
- Какую боль закрываем (сценарий использования, объём: этикеток в смену и т.д.)?

### Продукт и данные

- Что на этикетке (название, дата/время изготовления, срок годности до; единицы: часы/дни)?
- Правило расчёта срока годности (например: изготовление + срок из справочника, затем минус N минут)?
- Откуда справочник сроков: только редактирование в приложении или ещё импорт из файла (CSV, опционально xlsx)? Кнопки «Сохранить» / «Обновить справочник» — новые данные вступают в силу после нажатия.

### Платформа и этапы

- Этап 1: платформа (ПК: браузер + локальный сервис?), принтер (модель, USB/Bluetooth)?
- Этап 2: Telegram-бот, печать через ПК?
- Этап 3: мобильное приложение (Android), печать по Bluetooth?

### Голос и язык

- Язык: только русский (голос, названия, даты, команды)?
- Разбор фразы: парсер правил или ИИ по API? Приложение знает текущие дату/время; относительные формулировки («вчера 18:10») обрабатываются.

### Стек (Этап 1)

- Фронт: браузер (React или простой HTML/JS)? Web Speech API для голоса.
- Локальный сервис на ПК: Node.js (Express) или Python (FastAPI)?
- Печать: библиотека ESC/POS под выбранный язык; принтер Xprinter XP-365B (USB, затем при необходимости Bluetooth).
- Справочник: хранение (JSON/SQLite); импорт CSV (и опционально xlsx).

### Качество и сроки

- Покрытие тестами: обязательно; что именно тестируем (парсер, расчёт сроков, API справочника, печать — по возможности).
- Сроки: рабочий прототип за 1–2 дня; доработка до полного функционала — следующие несколько дней.

### Интерфейс и настройки

- Простой и понятный интерфейс; отдельный раздел «Настройки».
- Светлая и тёмная тема (переключатель в настройках).

## Как использовать

1. Пройти по блоку «Обязательные вопросы»: проверить, что все пункты имеют ответ (из чата, из требований, из допущений).

3. После доработки этого файла — обновить список вопросов и принципы при необходимости.
