# Размеры элементов: таблица «Список продуктов»

Отдельная запись для последующей проработки вариантов работы с шириной колонок и отступами.

---

## Текущий вариант (на момент сохранения)

**Режим:** колонки можно менять ползунками (ресайз вручную).

- **Где задаётся:** состояние в `client/src/App.jsx`: `shelfColWidths` (в rem), таблица с классом `shelf-table-resizable`, `<colgroup>` с ширинами из state.
- **Единицы:** rem (значения: product 20, expiry 7, actions 11).
- **Ограничения:** min/max при перетаскивании (product 10–45, expiry 5–12, actions 9–18 rem).
- **Стили:** `client/src/App.css` — блок «Таблица с ресайзом колонок», `.shelf-col-resizer`, классы `.shelf-product-col`, `.shelf-expiry-col`, `.shelf-actions-col`.
- **Отступы ячеек:** в `.shelf-table th, .shelf-table td` — `padding: 0.5rem 0.75rem`.

Ползунки — у правого края каждого заголовка колонки (Продукт, Срок, пустой заголовок над кнопками).

---

## Вариант для проработки: фиксированные размеры в пикселях

Чтобы колонки не двигались и не зависели от ресайза, можно перейти на фиксированные ширины и отступы в **px**.

**Пример значений (для обсуждения/подстройки):**

| Элемент              | Ширина / отступ |
|----------------------|------------------|
| Колонка «Продукт»    | 280px            |
| Колонка «Срок»       | 90px             |
| Колонка кнопок       | 140px            |
| Padding ячеек (th/td)| например 8px 12px |

**Что потребуется при переходе на этот вариант:**

1. Убрать из `App.jsx`: state `shelfColWidths`, `shelfResizingCol`, `shelfResizeStartRef`, `useEffect` ресайза, `startShelfResize`, `<colgroup>` и ползунки в `<th>`.
2. Задать таблице класс без ресайза (например `shelf-table-fixed`) и `table-layout: fixed`.
3. В CSS задать для колонок фиксированные `width` в px и при необходимости `padding` ячеек в px.
4. При необходимости подправить числа по факту (по картинке или по пожеланию).

---

## Заметки

- Размеры в rem зависят от размера шрифта корня (обычно 16px).
- Если позже понадобится сохранять выбранные пользователем ширины между сессиями — можно писать значения в `localStorage` при ресайзе и читать при открытии модалки.
