# Карта проекта DDLabel

Краткий обзор структуры проекта: важные папки, ключевые файлы, точки входа.

---

## Текущая структура

- **Корень проекта** — PLAN.md (план, роль, этапы, стек), PROJECT_MAP.md, PROJECT_LOG.md, PROMPT_PLAN.md (промт и вопросы для плана), README.md, LICENSE, .gitignore (Node.js).
- **`.cursor/rules/`** — правила чата, форматы, лог/карта, запрет секретов и т.д.
- **`.github/workflows/`** — ci.yml (CI при push в main; зелёная галочка у коммитов).
- **`docs/`** — справочные материалы: printer-xp365b.md (принтер XP-365B), **label-format.md** (формат этикетки 30×20 мм — название, даты ДД.ММ, время ЧЧ.ММ, символ ∞, две колонки).

**Этап 1 (в работе):**
- **`server/`** — Node.js (Express): `index.js` (API печати по готовым данным и по фразе, GET /api/printers, POST /api/parse), `labelBuilder.js` (шаблон этикетки по docs/label-format.md), `phraseParser.js` (разбор фразы: продукт, вчера/сегодня/позавчера, время ЧЧ:ММ), `shelfLife.js` (тестовый справочник сроков в часах, расчёт «срок до» минус 5 мин); ESC/POS через escpos и escpos-usb; в package.json override `usb: 1.8.8`. Запуск: `npm start`. Порт 3001.
- **`client/`** — React (Vite): поле ввода фразы, кнопки «Только разобрать», «Разобрать и напечатать», «Голос» (Web Speech API, ru-RU — распознанный текст подставляется в поле и отправляется на разбор и печать); вывод разобранных данных; блок «Печать тестовой этикетки»; прокси `/api` на сервер. Читаемость: карточка и поле ввода — явные цвета в App.css. Запуск: `npm run dev`. Интерфейс: http://localhost:3000 (port 3000, host: true).

---

## Стек и точки входа

- **План:** PLAN.md; промт для плана — PROMPT_PLAN.md.
- **Стек Этапа 1:** фронт — браузер **React**, Web Speech API; бэкенд — Node.js (Express); печать — ESC/POS (Xprinter XP-365B по USB).
- **Точки входа:** сервер — `server/index.js`; фронт — `client/` (Vite), вход — `index.html` → `src/main.jsx` → `App.jsx`.
