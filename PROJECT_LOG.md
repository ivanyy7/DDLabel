# Лог проекта DDLabel

История решений и шагов относительно плана. Краткие записи без кода.

## Ключевые шаги и история

- *Старт* — созданы PLAN.md (пока пустой), PROJECT_MAP.md, PROJECT_LOG.md. Идёт обсуждение будущего проекта перед написанием плана.
- Обсуждение завершено: цель (кафе, этикетки 30×20, до ~100 в смену), три этапа (ПК → Telegram → Android), справочник сроков (редактирование в приложении + загрузка из файла CSV/xlsx), правило «срок годности до» минус 5 минут, русский язык, принтер Xprinter XP-365B.
- План оформлен в PLAN.md: роль (мидл/сеньор), цель и контекст, справочник (отдельный пункт), этапы 1–2–3, стек, качество и сроки, критерии готовности.
- Создан PROMPT_PLAN.md: промт для составления плана, обязательные вопросы, роль.
- Выбран стек: Node.js (Express) для локального сервиса; зафиксировано в PLAN.md.
- Добавлены .gitignore (Node), CI (.github/workflows/ci.yml), все файлы (план, карта, лог, промт, правила) закоммичены и запушены на GitHub.
- Добавлен справочник по принтеру: docs/printer-xp365b.md (выжимка из руководства — драйверы, Android Print Label, USB/Bluetooth, характеристики, устранение неполадок). Карта обновлена.
- Фронт зафиксирован: React (удобство таблицы справочника и структуры с первого дня). PLAN.md и PROJECT_MAP.md обновлены.
- В PLAN.md добавлен порядок работ внутри Этапа 1: семь последовательных мини-этапов (каркас и печать → парсер и расчёт → голос → справочник → импорт из файла → настройки и тема → тесты и полировка).
- Зафиксированы формат этикетки (docs/label-format.md), встроенные шрифты принтера (конкретный шрифт/начертание — уточнит пользователь), ссылки на драйверы и ПО принтера (Google Drive) в docs/printer-xp365b.md.
- День 1, финал сессии — этап планирования завершён. Все решения зафиксированы в PLAN.md, PROJECT_MAP.md, docs/. Готовность к старту реализации Этапа 1.
- День 2 — мини-этап 1 (каркас и печать по шаблону): созданы server/ (Express, escpos, escpos-usb) и client/ (React, Vite). Реализован шаблон этикетки в labelBuilder.js по docs/label-format.md (название по центру, даты/время в две колонки, символ ∞). API POST /api/print принимает productName, madeAt, expiresAt; при отсутствии принтера возвращает 503. Фронт: кнопка «Печать тестовой этикетки» с тестовыми данными, прокси на сервер. README обновлён — как запускать сервер и фронт.
- День 2, финал сессии — мини-этап 1 закрыт и зафиксирован. Интерфейс переведён на порт 3000 (vite.config.js: port 3000, host: true для стабильного доступа). Читаемость: в карточке заданы явные цвета (тёмный текст на светлом фоне), чтобы без расширения Dark Reader текст был читаем. Проверено: при отключённом принтере корректно показывается «Принтер не найден». Записи внесены в LOG и MAP.

## Технические решения и грабли

- Шрифты: только встроенные принтера (ESC/POS); конкретный размер/начертание пользователь укажет позже.
- Фронт: React (таблица справочника, структура).
- Порядок работ: семь мини-этапов внутри Этапа 1 — выполнять последовательно, каждый с явным результатом.
- Интерфейс: порт 3000; в Vite задано host: true для надёжного доступа к localhost.
- Читаемость: в карточке явно заданы цвет текста (#1a1a1a) и фон, чтобы без расширений темы (Dark Reader и т.п.) текст оставался читаемым.
