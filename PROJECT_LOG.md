## Технические решения и грабли

- Шрифты: только встроенные принтера (ESC/POS); конкретный размер/начертание пользователь укажет позже.
- Фронт: React (таблица справочника, структура).
- Порядок работ: семь мини-этапов внутри Этапа 1 — выполнять последовательно, каждый с явным результатом.
- Интерфейс: порт 3000; в Vite задано host: true для надёжного доступа к localhost.
- Читаемость: в карточке и в поле ввода фразы явно заданы цвет текста (#1a1a1a) и светлый фон, чтобы без расширений темы текст оставался читаемым.
- Печать по USB (Windows): после установки драйвера Windows принтер «занят» системой; для DDLabel (escpos-usb) нужен Zadig — замена драйвера на WinUSB. Инструкция в docs/printer-xp365b.md.
- Ошибка «usb.on is not a function»: пакет usb (зависимость escpos-usb) в новых версиях несовместим с escpos-usb. Решение: в server/package.json задать "overrides": { "usb": "1.8.8" }, затем npm install.
- Мини-этап 2: парсер опирается на ключевые слова (вчера/сегодня/позавчера) и шаблон времени ЧЧ:ММ; продукт — остаток фразы после удаления этих частей. Справочник сроков пока захардкожен в shelfLife.js (часы); «срок годности до» = изготовление + срок − 5 минут.
- Мини-этап 3: голос — Web Speech API (SpeechRecognition/webkitSpeechRecognition), язык ru-RU. Поддержка в Chrome, Edge; при отсутствии API показывается сообщение «Голосовой ввод не поддерживается в этом браузере».
- Парсер дат: опорная дата «сегодня» берётся по московскому времени (Intl.DateTimeFormat, timeZone: Europe/Moscow); «N дня/дней назад» — по цифре или по словам один–десять. Парсер пока нестабилен — темы для доработки: надёжное распознавание «два дня назад» и поиск по дополнительным названиям (алиасам).
- Таблица «Список продуктов»: первый вариант — ширины колонок в state (rem) и ресайз ползунками; фиксированный вариант в px описан в docs/UI-размеры-таблица-продуктов.md. В текущей версии, после долгих попыток добиться аккуратного выравнивания столбца «Срок» и кнопок, перешли на адаптивную таблицу с процентными ширинами столбцов и небольшими @media-правками; важно учитывать, что изменение padding внутри flex-ячейки не двигает границу столбца при table-layout: fixed — для выравнивания нужно работать с ширинами колонок и общей шириной модалки.
